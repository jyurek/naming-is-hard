2. Be specific

If you can't name it, it's probably wrong

Don't be too general

SRP applies to names, too

Datum

If it has one function, it easier to name

Sandi Metz's rules

  1. Classes can be no longer than one hundred lines of code.
  2. Methods can be no longer than five lines of code.
  3. Pass no more than four parameters into a method. Hash options are parameters.
  4. Controllers can instantiate only one object. Therefore, views can only know about one instance variable and views should only send messages to that object (@object.collaborator.value is not allowed).

Small classes, small methods

That means many classes and methods

The parts become labelled for you

Best action is to extract methods

3. Refactoring and extraction reveal meaning

Learn how it works by extracting methods

Extracting methods lets you understand intent

Knowing intent lets you apply names

Document the system

Code can tell a story

Less specific to more specific as you go down

def has_attached_file(name, options = {})
  include InstanceMethods

  if attachment_definitions.nil?
    self.attachment_definitions = {}
  else
    self.attachment_definitions = self.attachment_definitions.dup
  end

  attachment_definitions[name] = Paperclip::AttachmentOptions.new(options)
  Paperclip.classes_with_attachments << self.name

  after_save :save_attached_files
  before_destroy :prepare_for_destroy
  after_destroy :destroy_attached_files

  define_paperclip_callbacks :post_process, :"#{name}_post_process"

  define_method name do |*args|
    a = attachment_for(name)
    (args.length > 0) ? a.to_s(args.first) : a
  end

  define_method "#{name}=" do |file|
    attachment_for(name).assign(file)
  end

  define_method "#{name}?" do
    attachment_for(name).file?
  end

  validates_each(name) do |record, attr, value|
    attachment = record.attachment_for(name)
    attachment.send(:flush_errors)
  end
end

4. Your names should make sense

Ruby isn't statically typed

A array called "users" should contain Users

Don't ever name a variable "x"

Try searching for "x"

Let your names document your code
